# archive-official-home

本仓库用于归档和发布多个独立的 HTML 页面。这些页面通常对应不同网站的主页或特定功能页面，以便于长期保存、离线查阅或快速部署。

## 项目简介

archive-official-home 是一个轻量级的 HTML 页面归档仓库。它不针对任何特定域名或网站，而是作为一个通用容器，收纳各类官方主页的静态副本。仓库中的每个页面均保持独立的 HTML 结构，不依赖外部动态服务，可直接在浏览器中打开使用。

## 目录结构

```
archive-official-home/
├── pages/               # 存放归档的 HTML 页面
│   ├── example-page-1.html
│   ├── example-page-2.html
│   └── ...
├── assets/              # 共享资源（CSS、JS、图片等）
│   ├── styles.css
│   └── scripts.js
├── README.md            # 本文件
└── LICENSE              # 许可证信息
```

- **pages/**：核心目录，每个 HTML 文件对应一个归档页面。文件名应简洁且能反映页面内容，例如 `login-page.html` 或 `dashboard-v2.html`。
- **assets/**：存放所有页面可能共享的样式、脚本或图片资源，避免重复引用。

## 页面归档说明

- **来源**：归档页面来源于公开可访问的网站主页，仅用于存档和参考目的。
- **格式**：每个页面均为纯 HTML 文件，必要时可包含内联样式或脚本，以保持独立可用性。
- **更新频率**：不保证实时同步，仅在有需要时手动更新归档版本。
- **兼容性**：页面在主流浏览器（Chrome、Firefox、Edge）中测试可用，但不保证与所有旧版本浏览器兼容。

## 维护说明

本仓库由维护者不定期更新。如果您希望提交新的归档页面或修改现有页面，请遵循以下原则：

1. 提交的 HTML 文件应放置于 `pages/` 目录下。
2. 文件名使用小写字母和连字符，如 `new-page.html`。
3. 若页面依赖外部资源（如图片、字体），请尽量使用相对路径引用 `assets/` 中的文件。
4. 提交前请确保页面能在无网络环境下正常打开（即不依赖在线 CDN 或 API）。
5. 欢迎通过 Issue 或 Pull Request 提出改进建议，但请注意本仓库不接受包含商业推广或恶意内容的请求。
